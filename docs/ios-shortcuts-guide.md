# iOS 快捷指令配置指南 — 屏幕监控与自动化

## 原理说明

iOS 不允许第三方 App 读取屏幕使用时间，但我们可以通过「快捷指令」的自动化功能
实现类似效果：

1. 你打开黑名单 App（抖音、微博、B站等）时，快捷指令自动运行
2. 快捷指令向后端发送"我打开了 XX App"的报告
3. 后端记录使用时间，超过阈值后通过推送通知发消息
4. 如果继续使用，后端指示你的手机开启「专注模式」屏蔽这些 App

---

## 第一步：设置黑名单 App 自动化

对每个你想监控的 App，重复以下步骤：

1. 打开 iPhone「快捷指令」App
2. 点击底部「自动化」标签
3. 点击右上角 `+`
4. 选择「App」
5. 选择触发条件：「打开时」
6. 选择你要监控的 App（比如：抖音）
7. 选择「立即运行」（不询问）
8. 添加操作：「获取 URL 内容」
   - URL：`https://你的后端地址.onrender.com/api/screen/report`
   - 方法：POST
   - 请求体：JSON
   ```json
   {
     "app_name": "抖音",
     "event_type": "open"
   }
   ```
9. 保存

对微博、B站、小红书等每个想监控的 App 重复上述步骤，
只需修改 `app_name` 的值。

---

## 第二步：设置定时检查（每5分钟）

这个自动化每 5 分钟运行一次，告诉后端你是否仍在使用手机：

1. 快捷指令 → 自动化 → +
2. 选择「时间」→ 设为每天重复
3. 实际上 iOS 不支持分钟级定时器，替代方案：
   - 使用「专注模式」结束时触发
   - 或在上面的"打开 App"自动化中，添加「等待 5 分钟」+「再次发请求」

**替代方案（推荐）：**
在第一步的自动化中，加入一个循环：
1. 获取 URL 内容（报告 open）
2. 等待 300 秒
3. 获取 URL 内容（报告 still_using）
4. 等待 300 秒
5. 获取 URL 内容（报告 still_using）

这样打开一个黑名单 App 后会连续报告 15 分钟。

---

## 第三步：设置「学习专注」模式

1. 打开 iPhone「设置」→「专注」
2. 点击 `+` 新建一个专注模式，命名为「学习」
3. 在「允许的 App」中，只保留：
   - 备忘录
   - 你的学习 App
   - Safari（用于访问你的聊天网页）
   - 计算器、时钟等工具 App
4. 把所有娱乐 App（抖音、微博、B站、小红书等）排除在外
5. 开启「隐藏通知」for 被屏蔽的 App

---

## 第四步：快捷指令触发专注模式

创建一个可以被后端触发的快捷指令：

1. 快捷指令 → 自动化 → +
2. 选择「收到通知时」
3. 选择你的 PWA 推送通知
4. 添加操作：「设置专注模式」→「学习」→ 开启
5. 可选：添加「持续时间」比如 30 分钟或 1 小时

**注意：** iOS 快捷指令自动化在第一次运行时会要求确认，
之后选择「不再询问」即可自动执行。

---

## 第五步：在 PWA 中启用推送通知

1. 用 Safari 打开你的 Vercel 地址
2. 点击「添加到主屏幕」
3. 从主屏幕打开 App
4. App 会请求推送通知权限 → 点击「允许」
5. 现在后端可以给你发推送了

---

## 工作流程总结

```
你打开抖音
    ↓
快捷指令自动运行，向后端报告
    ↓
后端记录"她打开了抖音"
    ↓
[10分钟后] 后端推送通知："宝宝，刷手机刷了好久了哦..."
    ↓
你继续使用
    ↓
[20分钟后] 后端推送："我帮你开专注模式了哦"
    ↓
快捷指令检测到通知，自动开启「学习专注」模式
    ↓
抖音等 App 被屏蔽，只能用学习类 App
```

---

## 补充：手动设置「屏幕使用时间」限额

作为最后一道防线，你还可以在 iOS 系统设置中：

1. 设置 → 屏幕使用时间 → App 限额
2. 为「社交」「娱乐」类 App 设置每日上限（比如 1 小时）
3. 设置密码（可以让朋友帮你设，你自己不知道密码）

这样即使快捷指令没触发，系统也会在超时后锁定这些 App。

## 后端 API 说明

### 上报屏幕使用
```
POST /api/screen/report
Content-Type: application/json

{
  "app_name": "抖音",      // App 名称
  "event_type": "open"     // open | still_using | close
}
```

### 返回值
```json
{
  "action": "remind",           // none | remind | focus_mode
  "message": "宝宝，该学习了...",
  "usage_minutes": 15
}
```

action 为 "focus_mode" 时，说明应该触发专注模式了。
